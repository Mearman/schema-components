## Start with a document

Given an OpenAPI 3.1 document like the one below, every story on this page is
driven by it — there is no separate setup per component:

<Source
    language="ts"
    code={`const petStore = {
    openapi: "3.1.0",
    info: { title: "Pet Store", version: "1.0.0" },
    paths: {
        "/pets": {
            get: {
                summary: "List pets",
                parameters: [
                    {
                        name: "limit",
                        in: "query",
                        required: false,
                        schema: { type: "integer", minimum: 1, maximum: 100 },
                    },
                    {
                        name: "status",
                        in: "query",
                        required: false,
                        schema: { type: "string", enum: ["available", "pending", "sold"] },
                    },
                ],
                responses: { "200": { description: "A list of pets" /* ... */ } },
            },
            post: {
                summary: "Create a pet",
                requestBody: { /* ... */ },
                responses: { "201": { description: "Created" } },
            },
        },
    },
    components: {
        schemas: {
            Pet: {
                type: "object",
                properties: {
                    id: { type: "string", format: "uuid" },
                    name: { type: "string" },
                    status: { type: "string", enum: ["available", "pending", "sold"] },
                },
                required: ["id", "name"],
            },
        },
    },
} as const;`}
/>

## Rendering a component schema

The simplest entry point is `SchemaComponent` with a JSON Pointer to a component
schema. The walker resolves the `$ref`, normalises any 3.0-era keywords
(`nullable`, `discriminator`), and renders the same way it would render a Zod
schema:

<Source
    language="tsx"
    code={`<SchemaComponent
    schema={petStore}
    ref="#/components/schemas/Pet"
    value={{ id: "pet_abc123", name: "Fido", status: "available" }}
/>`}
/>

<Canvas of={OpenApiStories.ComponentSchema} />

## Whole operations with `<ApiOperation>`

For a complete operation — parameters, request body, and every response — use
`ApiOperation`. It pulls everything for one `path` and `method` from the document
and renders headings, content-type badges, status codes, and the underlying
schemas in one go:

<Source
    language="tsx"
    code={`import { ApiOperation } from "schema-components/openapi/components";

<ApiOperation schema={petStore} path="/pets" method="post" />`}
/>

<Canvas of={OpenApiOperationStories.FullOperation} />

## Drilling in: parameters only

When you only want the parameter list — for a sidebar, a quick reference card, or
an embedded snippet — `ApiParameters` skips the rest:

<Source
    language="tsx"
    code={`import { ApiParameters } from "schema-components/openapi/components";

<ApiParameters schema={petStore} path="/pets" method="get" />`}
/>

<Canvas of={OpenApiOperationStories.ParametersOnly} />

## Drilling in: request body only

`ApiRequestBody` extracts the content schema for the chosen media type (defaults
to `application/json`) and renders it just like a top-level schema. Field
overrides are type-safe when the document is declared `as const`:

<Source
    language="tsx"
    code={`import { ApiRequestBody } from "schema-components/openapi/components";

<ApiRequestBody
    schema={petStore}
    path="/pets"
    method="post"
    fields={{ name: { description: "Pet name" } }}
/>`}
/>
