    `renderToHtml` emits semantic HTML strings from any schema — no React runtime,

## Why a separate HTML renderer?

The React renderer assumes a React tree, hooks, and (for editable mode) state. A
lot of real-world surfaces have none of those:

- Server-rendered pages where you only need the HTML markup.
- Email templates that must be a single string of valid HTML.
- Static site builds that emit markup at build time.
- Admin tools written in another framework that still want to render schemas
  consistently.

`renderToHtml` shares the same walker as the React path, so a schema renders to
the same shape (and the same `sc-*` class hooks) regardless of which output you
use.

## Static rendering

The simplest form: pass a schema and a value, get back a string of HTML:

<Source
    language="tsx"
    code={`import { renderToHtml } from "schema-components/html/renderToHtml";

const html = renderToHtml(userSchema, {
    value: { name: "Ada Lovelace", email: "ada@example.com", role: "admin" },
});`}

## Default stylesheet

The HTML output uses semantic markup with `sc-*` class hooks. A default
stylesheet is bundled at `schema-components/styles.css` (also published as
`schema-components/html/styles.css`). Link it for instant styling:

<Source
    language="html"
    code={`<link rel="stylesheet" href="/path/to/schema-components/styles.css" />
${"<!-- ...your rendered HTML here... -->"}`}
/>

## Streaming rendering

For large schemas or progressive page loads, `renderToHtmlChunks` yields an
iterable of string chunks that you can pipe straight into a response stream:

<Source
    language="tsx"
    code={`import { renderToHtmlChunks } from "schema-components/html/renderToHtmlStream";

for (const chunk of renderToHtmlChunks(largeSchema, { value: data })) {
    response.write(chunk);
}`}
/>

<Canvas of={HtmlStreamingStories.LargeSchemaChunks} />

## Custom resolvers

The HTML renderer accepts an `HtmlResolver` — the same shape as
`ComponentResolver` from the React side, but each function returns a `string`
instead of a `ReactElement`. Override individual schema types and the rest fall
back to the default renderers:

<Source
    language="tsx"
    code={`import {
    renderToHtml,
    type HtmlResolver,
} from "schema-components/html/renderToHtml";

const highlightResolver: HtmlResolver = {
    string: (props) => {
        if (props.readOnly) {
            const text = typeof props.value === "string" ? props.value : "";
            return \`<span class="sc-value sc-value--highlight">\${text}</span>\`;
        }
        return renderToHtml(z.string(), { value: props.value });
    },
};

const html = renderToHtml(cardSchema, {
    value: cardData,
    readOnly: true,
    resolver: highlightResolver,
});`}
/>
